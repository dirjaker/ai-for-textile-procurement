# 第三章：什么是 AI Agent？

> 从"对话"到"行动"的进化

---

## 一、从一个场景开始

**传统 AI（ChatGPT 对话）：**

```
你：帮我分析这 10 家供应商的数据

AI：好的，请把数据发给我

你：[粘贴数据]

AI：根据数据分析，供应商 A 质量最好，供应商 B 价格最低...

你：帮我做个表格

AI：好的，这是表格...

你：保存到 Excel

AI：抱歉，我无法操作你的电脑，你需要自己复制粘贴到 Excel
```

**AI Agent：**

```
你：帮我分析供应商数据，生成评估报告，保存到桌面

Agent：好的，我来处理
  ✅ 已读取数据文件
  ✅ 正在分析...
  ✅ 生成评估报告
  ✅ 已保存到桌面/supplier_report.xlsx
  ✅ 完成！报告包含详细分析和建议
```

**区别在哪？**
- 传统 AI：只能聊天，不能"做事"
- AI Agent：能聊天，还能调用工具、执行任务

## 二、AI Agent 的定义

### 2.1 简单理解

**AI Agent** = 大语言模型 + 工具使用 + 自主决策

**类比：**
- 大语言模型 = 聪明的大脑（能思考、能说话）
- AI Agent = 有了手和脚的聪明大脑（能思考、能说话、还能做事）

### 2.2 核心组件

```
┌─────────────────────────────────────────────────────┐
│                    AI Agent                          │
│                                                     │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐         │
│  │  大脑   │    │  工具   │    │  记忆   │         │
│  │ (LLM)  │ +  │ (Tools) │ +  │(Memory) │         │
│  └────┬────┘    └────┬────┘    └────┬────┘         │
│       │              │              │               │
│       └──────────────┼──────────────┘               │
│                      │                              │
│                      ▼                              │
│              ┌───────────────┐                      │
│              │   规划引擎    │                      │
│              │  (Planning)   │                      │
│              └───────────────┘                      │
│                      │                              │
│                      ▼                              │
│              ┌───────────────┐                      │
│              │   执行引擎    │                      │
│              │ (Execution)   │                      │
│              └───────────────┘                      │
└─────────────────────────────────────────────────────┘
```

### 2.3 核心能力

| 能力 | 说明 | 举例 |
|------|------|------|
| **理解任务** | 理解用户意图 | "分析供应商" → 知道要做什么 |
| **规划步骤** | 将大任务分解 | 分析 → 收集数据 → 计算 → 生成报告 |
| **使用工具** | 调用各种工具 | 读文件、写代码、搜索网页、发邮件 |
| **记忆能力** | 记住上下文 | 记住之前的对话和任务 |
| **自我反思** | 检查结果，必要时重试 | 发现数据有误 → 重新分析 |
| **自主决策** | 根据情况做判断 | 数据不足 → 自动补充收集 |

## 三、AI Agent 的工作流程

### 3.1 基本流程

```
用户需求
    │
    ▼
┌───────────────┐
│   理解任务    │ ← Agent 分析用户想要什么
└───────┬───────┘
        │
        ▼
┌───────────────┐
│   制定计划    │ ← Agent 拆分成多个步骤
└───────┬───────┘
        │
        ▼
┌───────────────┐
│   执行步骤    │ ← Agent 一步一步执行
│   (循环)      │
└───────┬───────┘
        │
        ├── 调用工具：读文件、搜索、写代码...
        ├── 获取结果
        ├── 检查结果是否正确
        └── 如果不对，重新尝试
        │
        ▼
┌───────────────┐
│   返回结果    │ ← Agent 给用户最终结果
└───────────────┘
```

### 3.2 实际例子

**用户需求：** "帮我分析上个月的采购数据，找出成本最高的供应商，并生成报告"

**Agent 的思考和执行过程：**

```
Step 1: 理解任务
  - 需要分析采购数据
  - 找出成本最高的供应商
  - 生成报告

Step 2: 制定计划
  1. 查找采购数据文件
  2. 读取数据
  3. 按供应商分组统计
  4. 找出成本最高的供应商
  5. 生成分析报告
  6. 保存报告

Step 3: 执行
  [调用工具] 搜索文件 → 找到 purchase_2024.xlsx
  [调用工具] 读取 Excel → 获取数据
  [调用工具] 运行 Python 脚本 → 统计分析
  [调用工具] 生成报告 → 创建 report.docx
  [调用工具] 保存文件 → 保存到桌面

Step 4: 返回结果
  "已完成分析！
   成本最高的供应商是：XX 公司（占比 35%）
   报告已保存到桌面/purchase_report.docx"
```

## 四、AI Agent 能用的工具

### 4.1 常见工具类型

| 工具类型 | 功能 | 举例 |
|---------|------|------|
| **文件操作** | 读写文件 | 读取 Excel、写入 Word |
| **代码执行** | 运行程序 | Python 脚本、数据分析 |
| **网络访问** | 搜索网页 | 查找供应商信息、市场行情 |
| **API 调用** | 调用服务 | 发邮件、查天气、查汇率 |
| **数据库** | 查询数据 | 查 ERP 数据、库存信息 |
| **浏览器** | 网页操作 | 自动填表、采集信息 |

### 4.2 采购场景的工具应用

| 任务 | 需要的工具 |
|------|-----------|
| 分析采购数据 | 文件读取 + 代码执行 + 可视化 |
| 收集供应商信息 | 网络搜索 + 信息提取 |
| 生成报告 | 文件读取 + 文档生成 |
| 发送询价邮件 | 邮件 API + 模板生成 |
| 监控价格 | 网络爬虫 + 数据分析 |
| 审查合同 | 文档读取 + 条款分析 |

## 五、主流 AI Agent 工具

### 5.1 Codex (OpenAI)

**是什么：** OpenAI 推出的代码生成和执行 Agent

**特点：**
- 强大的代码生成能力
- 支持多种编程语言
- 可以运行在本地环境

**适用场景：**
- 数据处理和分析
- 自动化脚本编写
- Excel/文件操作

**学习难度：** ⭐⭐⭐ 中等

### 5.2 Claude Code (Anthropic)

**是什么：** Anthropic 推出的 AI 编程助手

**特点：**
- 强大的代码理解能力
- 安全性高
- 擅长代码重构

**适用场景：**
- 复杂项目开发
- 代码审查和优化
- 安全要求高的场景

**学习难度：** ⭐⭐⭐ 中等

### 5.3 Cursor

**是什么：** AI 增强的代码编辑器

**特点：**
- 实时代码建议
- 集成 AI 对话
- 支持多种语言

**适用场景：**
- 编写和调试代码
- 学习编程
- 快速原型开发

**学习难度：** ⭐⭐ 较低

### 5.4 GitHub Copilot

**是什么：** GitHub 官方 AI 编程助手

**特点：**
- 实时代码补全
- 与 VS Code 深度集成
- 学习能力强

**适用场景：**
- 日常编程任务
- 代码补全和建议
- 快速开发

**学习难度：** ⭐⭐ 较低

### 5.5 通用 Agent 平台

| 平台 | 特点 | 适用场景 |
|------|------|---------|
| **Hermes Agent** | 多平台、多工具、易用 | 通用任务自动化 |
| **AutoGPT** | 全自动、自主执行 | 复杂研究任务 |
| **MetaGPT** | 多角色协作 | 项目管理 |
| **ChatGPT Plus** | 内置工具、使用简单 | 日常任务 |

### 5.6 如何选择？

| 你的需求 | 推荐工具 | 理由 |
|---------|---------|------|
| **学习入门** | ChatGPT Plus | 最简单，有内置工具 |
| **写代码** | Cursor / Copilot | 专业代码编辑 |
| **数据分析** | Codex | 数据处理能力强 |
| **通用任务** | Hermes Agent | 功能全面，易上手 |
| **复杂项目** | Claude Code | 理解能力最强 |

## 六、AI Agent vs 传统 AI

### 6.1 对比表

| 维度 | 传统 AI（对话） | AI Agent |
|------|----------------|----------|
| **交互方式** | 只能对话 | 对话 + 执行任务 |
| **工具使用** | ❌ 不能用工具 | ✅ 可以调用各种工具 |
| **任务执行** | ❌ 只能给建议 | ✅ 可以直接执行 |
| **多步骤** | ❌ 每步都要手动 | ✅ 自动完成多步骤 |
| **文件操作** | ❌ 不能读写文件 | ✅ 可以读写各种文件 |
| **代码运行** | ❌ 只能生成代码 | ✅ 可以运行代码 |
| **自主性** | 低（需要人指导每一步） | 高（可以自主规划执行） |

### 6.2 使用场景对比

**场景 1：分析 Excel 数据**

传统 AI：
```
你：这个 Excel 里有什么数据？
AI：请把数据内容发给我
你：[粘贴前几行]
AI：数据显示了采购记录...
你：帮我算一下总额
AI：需要你提供完整数据...
（来回很多次）
```

AI Agent：
```
你：帮我分析这个 Excel，按供应商统计总额，生成图表

Agent：
  ✅ 读取文件
  ✅ 分析数据
  ✅ 生成统计表
  ✅ 创建图表
  ✅ 保存结果
  "已完成！总额最高的供应商是 XX 公司，图表已保存"
```

传统 AI：
```
你：帮我查一下 XX 公司的信息
AI：抱歉，我无法访问网络，建议你去天眼查查询
```

AI Agent：
```
你：帮我收集 XX 公司的信息

Agent：
  ✅ 搜索网络...
  ✅ 查找工商信息...
  ✅ 查找新闻报道...
```

## 七、AI Agent 的局限性

### 7.1 仍然会出错

- 工具调用可能失败
- 理解可能有偏差
- 结果需要人工审核

### 7.2 需要明确的指令

- 模糊的指令会导致错误的结果
- 需要提供足够的上下文

### 7.3 复杂任务仍需人工

- 需要创意的工作
- 需要情感判断的工作
- 需要最终决策的工作

### 7.4 安全和隐私

- 需要授权才能操作
- 敏感数据要注意保护
- 重要操作要有确认机制
