# 第二章：什么是大语言模型（LLM）？

> 深入了解 ChatGPT 背后的技术

---

## 一、从一个对话开始

**你：** "帮我写一封给供应商的邮件，要求降价 5%"

**AI：**
```
尊敬的[供应商名称]：

您好！

感谢贵司一直以来的优质服务和合作支持。

近期，我们注意到市场原材料价格有所下调，同时我司采购量持续增长。
基于长期合作的良好关系，我们希望就产品价格进行友好协商。

具体建议如下：
1. 自[日期]起，整体采购价格下调 5%
2. 我们承诺保持现有采购量，并在条件合适时增加订单
3. 希望建立季度价格回顾机制，实现双赢

期待贵司的积极回复，我们可以安排会议进一步讨论。

此致
敬礼
[你的名字]
```

**这个 AI 就是大语言模型（Large Language Model，简称 LLM）**

## 二、大语言模型是什么？

### 2.1 简单定义

**大语言模型** = 用海量文本训练的、能够理解和生成人类语言的 AI 模型

**三个关键词：**
- **大**：模型参数量巨大（几十亿到几万亿）
- **语言**：专门处理人类语言
- **模型**：训练出来的"知识库"

### 2.2 通俗理解

**把大语言模型想象成一个读过全世界所有书的"超级学霸"：**

```
它读过的书：
├── 网页（数十亿个）
├── 书籍（数百万本）
├── 论文（数千万篇）
├── 代码（数十亿行）
└── 新闻、论坛、博客...
```

**它能做什么：**
- ✅ 回答各种问题
- ✅ 写文章、邮件、报告
- ✅ 翻译各种语言
- ✅ 总结长篇文章
- ✅ 写代码
- ✅ 数学推理

### 2.3 工作原理（极简版）

**第一步：训练（学习阶段）**
```
输入：互联网上的海量文本
      ↓
模型任务：预测下一个词
      ↓
举例："今天天气____"
      模型学习："真好"、"不错"、"很热"
      ↓
重复数十亿次，模型学会语言规律
```

**第二步：推理（使用阶段）**
```
用户输入： "帮我写一封邮件"
      ↓
模型预测： 根据上下文，预测最可能的下一个词
      ↓
一个词一个词生成，直到完成回答
```

### 2.4 "大"在哪里？

| 维度 | 说明 | 规模 |
|------|------|------|
| **参数量** | 模型内部的可调节参数 | GPT-3: 1750 亿参数 |
| **训练数据** | 用于训练的文本数据 | 数万亿个词 |
| **计算资源** | 训练所需的算力 | 数千块 GPU，训练数月 |
| **成本** | 训练费用 | 数百万到数千万美元 |

**GPT-4 的训练成本估计：1 亿美元**

```
  └───────────┴───────────┴───────────┴───────────┴───────────┘
                    基础架构 → 能力提升 → 走向大众
```

### 3.2 关键里程碑

**2017 年：Transformer 诞生**
- Google 发表论文《Attention is All You Need》
- 奠定了现代大语言模型的基础架构
- 解决了处理长文本的难题

**2018 年：BERT 发布**
- Google 推出
- 首次在多项 NLP 任务上超越人类
- 开启了预训练模型时代

**2020 年：GPT-3 发布**
- OpenAI 推出
- 1750 亿参数，当时最大模型
- 展示了强大的语言生成能力

**2022 年：ChatGPT 发布**
- 基于 GPT-3.5 微调
- 第一个面向大众的 AI 对话产品
- 2 个月用户破亿，引发 AI 热潮

**2023 年：GPT-4 发布**
- 多模态能力（能看图）
- 推理能力大幅提升
- 引领 AI 应用爆发

**2024 年：百花齐放**
- Claude 3、Gemini、DeepSeek 等涌现
- 开源模型快速发展
- Agent 应用开始流行

## 四、大语言模型的核心能力

### 4.1 文本生成

**场景举例：**
- 写邮件、报告、方案
- 生成产品描述
- 撰写新闻稿

**示例 Prompt：**
```

1. 本月采购完成情况
2. 主要供应商表现
3. 成本控制成果
4. 下月工作计划
```

### 4.2 文本理解与总结

**场景举例：**
- 总结合同要点
- 提取报告关键信息
- 分析客户反馈

**示例 Prompt：**
```
请总结以下合同的关键条款，用表格形式展示：
[粘贴合同内容]

需要提取的信息：
1. 产品规格
2. 价格条款
3. 交货期
4. 付款方式
5. 违约责任
```

### 4.3 信息提取

**场景举例：**
- 从询价单中提取参数
- 从简历中提取关键信息
- 从报告中提取数据

**示例 Prompt：**
```
请从以下询价单中提取关键信息：

[粘贴询价单]

输出格式：

  "product": "产品名称",
  "spec": "规格参数",

  "delivery_date": "交货日期",
  "quality_requirements": "质量要求"
```

### 4.4 问答与推理

**场景举例：**
- 解答专业知识问题
- 分析原因和影响
- 提供建议和方案

**示例 Prompt：**
```

```

### 4.5 代码生成

**场景举例：**
- 生成 Excel 公式
- 写数据处理脚本
- 自动化任务

**示例 Prompt：**
```

1. 读取 Excel 文件中的采购数据
2. 按供应商分组统计金额
3. 生成柱状图

```

### 4.6 翻译

**场景举例：**
- 翻译外文合同
- 翻译技术文档
- 多语言沟通

## 五、大语言模型的局限性

### 5.1 幻觉问题（最重要！）

**什么是幻觉？**
AI 生成的内容看起来很专业、很真实，但其实是编造的。

**例子：**
```
你问： "XX 公司的注册资本是多少？"

AI 答： "根据工商信息，XX 公司注册资本 5000 万元，成立于 2015 年。"

实际： AI 编造了这个数字！XX 公司可能不存在，或者数字完全错误。
```

**为什么会这样？**
- 模型是预测"最可能的下一个词"
- 不是真正"理解"或"知道"
- 当没有足够信息时，会"编造"一个看起来合理的答案

**如何应对：**
1. ✅ 重要信息一定要人工核实
2. ✅ 要求 AI 说明信息来源
3. ✅ 多个信息源交叉验证
4. ✅ 不要问 AI 不可能知道的事

### 5.2 知识截止

**问题：** AI 的知识只到训练数据的截止日期

**例子：**
- 如果 AI 是 2024 年训练的
- 你问 2025 年的事，它可能不知道或给出错误信息

**如何应对：**
- 涉及最新信息时，结合网络搜索
- 注意 AI 回答中的时间表述

### 5.3 缺乏实时信息

**问题：** AI 无法获取当前信息

**AI 不知道的事：**
- ❌ 今天的棉花价格
- ❌ 某家公司的最新财务数据
- ❌ 最新的行业新闻
- ❌ 你的邮件内容

**如何应对：**
- 需要实时信息时，使用能联网的 AI 工具
- 或者自己提供最新数据给 AI 分析

### 5.4 推理能力有限

**问题：** 复杂逻辑推理可能出错

**容易出错的情况：**
- 多步骤数学计算
- 复杂的因果关系
- 需要专业知识深度推理

**如何应对：**
- 将复杂问题分解成小问题
- 让 AI 分步骤展示推理过程
- 关键计算结果要人工验证

### 5.5 缺乏常识判断

**问题：** AI 可能不理解现实世界的"常识"

**例子：**
- AI 可能会建议"把工厂搬到月球上来降低成本"
- 或者"让 100 个人同时操作一台机器"

**如何应对：**
- 用常识判断 AI 的建议是否可行
- 提供足够的背景信息

## 七、如何开始使用？

### 7.1 注册账号

**推荐入门：**
1. **ChatGPT**：https://chat.openai.com（需科学上网）
2. **DeepSeek**：https://chat.deepseek.com（国内直接访问）
3. **通义千问**：https://tongyi.aliyun.com（国内直接访问）
4. **文心一言**：https://yiyan.baidu.com（国内直接访问）

### 7.2 基本使用技巧

**好的提问方式：**
```
明确角色 + 具体任务 + 充分上下文 + 期望格式

例如：
"你是一个有 10 年经验的坯布采购专家。
请分析纯棉 40s×40s 133×72 坯布的价格是否合理。
当前报价：8.5 元/米，历史价格：8-9 元/米，棉花价格近期下跌 5%。
请用表格形式对比分析。"
```

**避免的提问方式：**
```
❌ "这个价格合理吗？" （太模糊）
❌ "帮我分析一下" （不知道分析什么）
❌ "写个报告" （没有具体内容）
```

## 八、总结

### 关键要点

1. **大语言模型是什么**：用海量文本训练的 AI，能理解和生成语言

2. **核心能力**：
   - 文本生成（写邮件、报告）
   - 文本理解（总结、分析）
   - 信息提取
   - 问答推理

3. **重要局限**：
   - 幻觉问题（会编造信息）
   - 知识截止
   - 缺乏实时信息
   - 需要人工审核

4. **选择建议**：
   - 国内用户首选 DeepSeek 或通义千问
   - 需要最强能力选 GPT-4 或 Claude 3
   - 重视数据安全选开源模型本地部署
