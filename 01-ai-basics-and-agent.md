## 1. AI 基础概念

### 1.1 什么是人工智能 (AI)

人工智能是让计算机模拟人类智能的技术，包括：
- **学习**：从数据中获取知识
- **推理**：基于知识做出判断
- **自我纠正**：不断优化结果

### 1.2 机器学习 (Machine Learning)

机器学习是 AI 的核心分支，让计算机从数据中自动学习规律。

**常见类型：**
- **监督学习**：用标注数据训练（如：预测坯布价格）
- **无监督学习**：发现数据中的模式（如：供应商聚类分析）
- **强化学习**：通过试错优化策略（如：采购决策优化）

### 1.3 深度学习 (Deep Learning)

使用多层神经网络处理复杂数据，擅长：
- 图像识别（如：坯布瑕疵检测）
- 自然语言处理（如：合同条款分析）
- 语音识别（如：会议记录转写）

### 1.4 核心术语速查

| 术语 | 解释 | 采购场景举例 |
|------|------|-------------|
| **模型** | AI 学习后形成的"知识库" | 价格预测模型 |
| **训练** | 用数据教会 AI 的过程 | 用历史订单训练采购模型 |
| **推理** | AI 根据输入给出结果 | 输入规格，输出建议供应商 |
| **微调** | 在通用模型上针对特定领域优化 | 用纺织行业数据微调通用模型 |
| **Token** | AI 处理文本的基本单位 | 合同文本被拆分成 token 处理 |
| **幻觉** | AI 生成看似正确但错误的内容 | AI 编造不存在的供应商信息 |

## 2. 大语言模型 (LLM)

### 2.1 什么是大语言模型

大语言模型（Large Language Model）是基于海量文本训练的 AI 模型，能够理解和生成人类语言。

**代表模型：**
- **GPT-4** (OpenAI) - 最知名的通用大模型
- **Claude** (Anthropic) - 安全性高的对话模型
- **DeepSeek** - 国产高性能模型
- **通义千问** (阿里) - 中文能力强

### 2.2 LLM 能做什么

| 能力 | 采购场景应用 |
|------|-------------|
| **文本生成** | 撰写采购报告、邮件回复 |
| **文本摘要** | 快速阅读长篇合同、供应商资料 |
| **信息提取** | 从询价单中提取关键参数 |
| **翻译** | 处理外贸采购的英文资料 |
| **问答** | 解答纺织专业知识、采购流程问题 |
| **代码生成** | 生成 Excel 公式、数据处理脚本 |

### 2.3 LLM 的局限性

- **知识截止**：模型只"知道"训练数据截止前的信息
- **幻觉问题**：可能生成虚假信息，需要人工核实
- **缺乏实时性**：无法获取最新市场价格、库存信息
- **专业深度**：通用模型对细分领域（如坯布采购）理解有限

## 3. AI Agent 详解

### 3.1 什么是 AI Agent

AI Agent 是能够自主执行任务的智能体，不只是对话，还能：
- 调用工具（浏览器、文件系统、API）
- 执行多步骤任务
- 根据反馈调整策略

**类比：**
- 普通 AI = 智能顾问（只能给建议）
- AI Agent = 智能助手（能帮你做事）

### 3.2 主流 AI Agent 工具

#### 3.2.1 Codex (OpenAI)

**特点：**
- 专注于代码生成和执行
- 可以运行在本地环境
- 支持多编程语言

**适用场景：**
- 生成数据处理脚本
- 自动化 Excel 操作
- 构建简单的采购管理系统

#### 3.2.2 Cursor

**特点：**
- AI 增强的代码编辑器
- 实时代码建议和补全
- 支持多种编程语言

**适用场景：**
- 编写采购数据分析脚本
- 开发小型采购工具
- 学习编程的入门工具

#### 3.2.3 Claude Code

**特点：**
- Anthropic 推出的 AI 编程助手
- 强大的代码理解能力
- 注重安全性

**适用场景：**
- 复杂的代码重构
- 安全的代码审查
- 大型项目的开发辅助

#### 3.2.4 GitHub Copilot

**特点：**
- GitHub 官方 AI 编程助手
- 与 VS Code 深度集成
- 实时代码补全

**适用场景：**
- 日常编程任务
- 代码学习和参考
- 快速原型开发

#### 3.2.5 通用 AI Agent

**特点：**
- 不限于代码，可处理各种任务
- 支持文件操作、网页浏览、API 调用
- 多步骤任务自动化

**适用场景：**
- 自动化采购流程
- 供应商信息收集
- 市场分析报告生成

### 3.3 Agent 工作原理

```
用户需求 → Agent 理解 → 制定计划 → 调用工具 → 执行任务 → 返回结果
     ↑                                                    ↓
     ←←←←←←←←←←←← 反馈调整 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

**核心组件：**
1. **规划能力**：将大任务拆分成小步骤
2. **工具使用**：调用浏览器、文件系统、API 等
3. **记忆系统**：记住之前的对话和任务
4. **反思机制**：检查结果，必要时重新尝试

### 3.4 如何选择 Agent 工具

| 需求 | 推荐工具 | 理由 |
|------|---------|------|
| 写代码 | Cursor / Copilot | 专业代码编辑 |
| 数据分析 | Codex | 强大的数据处理能力 |
| 通用任务 | Claude / ChatGPT | 灵活的对话能力 |
| 复杂项目 | Claude Code | 代码理解和重构能力 |
| 学习编程 | Cursor | 实时反馈，学习友好 |

## 4. Prompt Engineering

### 4.1 什么是 Prompt Engineering

Prompt Engineering 是设计有效指令（Prompt）以获得最佳 AI 输出的技术。

**核心理念：** 清晰、具体、有上下文的指令 → 更好的结果

### 4.2 Prompt 设计原则

#### 原则一：明确角色和任务

**差：** "帮我分析一下这个供应商"

**好：** "你是一个有 10 年经验的坯布采购专家。请分析以下供应商的优缺点，并给出是否建议合作的意见：[供应商信息]"

#### 原则二：提供足够上下文

**差：** "这个价格合理吗？"

**好：** "我正在采购 100% 纯棉坯布，规格 40s×40s，133×72，幅宽 63 英寸。供应商报价 8.5 元/米，订单量 5000 米。请分析这个价格是否合理，并说明理由。"

#### 原则三：指定输出格式

**差：** "总结一下这个合同"

**好：** "请用以下格式总结合同要点：
1. 产品规格
2. 价格条款
3. 交货期
4. 付款方式
5. 质量标准
6. 违约责任"

#### 原则四：分步骤思考

**差：** "怎么降低采购成本？"

**好：** "请按以下步骤分析如何降低坯布采购成本：
1. 首先，分析当前成本构成（原料、加工、运输、仓储）
2. 然后，识别主要成本驱动因素
3. 接着，针对每个因素提出 2-3 个优化方案
4. 最后，评估每个方案的可行性和预期节省"

### 4.3 采购场景 Prompt 模板

#### 供应商评估

```
你是一个资深坯布采购专家。请评估以下供应商：

**供应商信息：**
- 公司名称：[名称]
- 主营产品：[产品]
- 合作历史：[年限/订单数]
- 价格水平：[与市场对比]
- 质量表现：[合格率/投诉率]
- 交货表现：[准时率]

**请从以下维度评分（1-10分）并说明理由：**
1. 产品质量
2. 价格竞争力
3. 交货可靠性
4. 沟通响应
5. 技术支持

**给出综合评估和合作建议。**
```

#### 价格分析

```
你是坯布市场价格分析专家。请分析以下数据：

**历史价格数据：**
[粘贴价格数据]

**请分析：**
1. 价格趋势（上涨/下跌/平稳）
2. 季节性规律
3. 影响因素（原料、汇率、政策等）
4. 未来 3 个月价格预测
5. 采购时机建议

**请用图表和数据支撑你的分析。**
```

#### 合同审查

```
你是一个纺织行业合同审查专家。请审查以下采购合同条款：

**合同内容：**
[粘贴合同条款]

**请检查：**
1. 是否存在对我方不利的条款
2. 质量标准是否明确
3. 交货期是否合理
4. 付款条件是否公平
5. 违约责任是否对等
6. 是否有遗漏的重要条款

**请逐条分析并给出修改建议。**
```

### 4.4 Prompt 优化技巧

1. **迭代优化**：第一次结果不满意，继续追问优化
2. **示例引导**：给出期望输出的示例
3. **限制范围**：明确告诉 AI 不要做什么
4. **验证要求**：要求 AI 说明推理过程和依据来源

## 5. AI 在采购中的应用

### 5.1 供应商管理

#### 5.1.1 供应商信息收集

**AI 工具：** ChatGPT / Claude / 通用 Agent

**应用场景：**
- 自动收集供应商背景信息
- 分析供应商财务状况
- 监控行业新闻和风险事件

**示例 Prompt：**
```
请帮我收集以下供应商的公开信息：
- 公司名称：[名称]
- 所在地区：[地区]
- 主营产品：[产品]

需要了解：
1. 公司规模（员工数、产能）
2. 主要客户
3. 行业口碑
4. 近期新闻
5. 潜在风险
```

#### 5.1.2 供应商评估模型

**AI 工具：** Codex / Python Agent

**应用场景：**
- 建立供应商评分体系
- 自动生成评估报告
- 供应商分级管理

**示例流程：**
```
输入：供应商历史数据（质量、价格、交货、服务）
↓
AI 处理：计算综合得分、识别优劣势
↓
输出：供应商评级、改进建议、合作策略
```

### 5.2 价格管理

#### 5.2.1 价格预测

**AI 工具：** Python + 机器学习库

**应用场景：**
- 预测坯布价格走势
- 识别最佳采购时机
- 评估价格风险

**数据来源：**
- 历史采购价格
- 原材料价格（棉花、化纤）
- 汇率变动
- 季节性因素

#### 5.2.2 成本分析

**AI 工具：** ChatGPT + Excel

**应用场景：**
- 分析成本构成
- 识别降本机会
- 对比不同方案成本

### 5.3 合同管理

#### 5.3.1 合同生成

**AI 工具：** ChatGPT / Claude

**应用场景：**
- 根据谈判结果生成合同初稿
- 标准化合同模板
- 翻译外文合同

#### 5.3.2 合同审查

**AI 工具：** Claude / 专业合同 AI

**应用场景：**
- 审查合同条款
- 识别风险条款
- 提供修改建议

### 5.4 数据分析

#### 5.4.1 采购数据分析

**AI 工具：** Codex / Python Agent

**应用场景：**
- 分析采购历史数据
- 识别采购模式和趋势
- 生成可视化报告

**示例任务：**
```
请分析过去 12 个月的采购数据：
1. 各品类采购金额占比
2. 月度采购趋势
3. 供应商集中度
4. 价格波动情况
5. 采购效率指标（订单周期、准时率等）
```

#### 5.4.2 市场情报

**AI 工具：** 通用 Agent + 网络搜索

**应用场景：**
- 监控行业动态
- 收集竞争对手信息
- 分析市场趋势

### 5.5 自动化流程

#### 5.5.1 询价自动化

**AI 工具：** Codex / Python Agent

**应用场景：**
- 自动生成询价单
- 批量发送询价邮件
- 收集整理报价

#### 5.5.2 报告自动化

**AI 工具：** 通用 Agent

**应用场景：**
- 自动生成周报/月报
- 更新供应商档案
- 维护采购台账

### 5.6 实际案例

#### 案例一：坯布价格预测

**背景：** 采购经理需要预测未来 3 个月的坯布价格

**步骤：**
1. 收集历史价格数据（2 年）
2. 使用 Python 训练时间序列模型
3. 考虑棉花价格、汇率等外部因素
4. 生成价格预测报告

**结果：** 预测准确率 85%，帮助公司在低价期采购，节省 12% 成本

#### 案例二：供应商风险监控

**背景：** 需要监控 50 家供应商的风险状态

**步骤：**
1. 设置 AI Agent 定期收集供应商新闻
2. 分析负面新闻并评估风险等级
3. 自动生成风险预警报告
4. 推送到采购团队工作群

**结果：** 提前发现 3 家供应商的潜在风险，避免了供应链中断

## 总结

AI 不是要取代采购经理，而是成为采购经理的"超级助手"：

1. **信息收集**：AI 帮你快速收集和整理信息
2. **数据分析**：AI 帮你处理复杂数据，发现规律
3. **决策支持**：AI 帮你分析选项，提供参考建议
4. **流程自动化**：AI 帮你处理重复性工作，释放时间

**关键心态：**
- 保持好奇心，勇于尝试新工具
- 批判性思维，不盲信 AI 结果
- 持续学习，跟上技术发展
- 结合经验，发挥人的判断力
